01 — À propos

Mon parcours

Après un bac général avec les spécialités Physique-Chimie et NSI, j'ai choisi de plonger dans le développement en intégrant un BTS SIO option SLAM puis un Bachelor chez EPSI.

En alternance chez Cotranet, j'ai travaillé sur une vraie application Java/Vue.js en production : système de release notes, documentation API avec Swagger, tickets Mantis.

Mon objectif est de devenir développeur. Ce qui m’intéresse surtout, c’est de travailler sur des projets concrets : comprendre un besoin, concevoir une solution et la faire fonctionner de bout en bout. J’apprécie particulièrement développement web, interfaces, backend, résolution de bugs, parce que ça me permet de voir directement l’impact de ce que je produis.

11 WatchHub — Instagram DM pour Galaxy Watch
TypeScript Expo Python FastAPI
+

Contexte

Application permettant d'accéder aux messages Instagram (DMs) depuis une Galaxy Watch 7 (Wear OS), avec un backend Python faisant office de proxy Instagram via instagrapi.

Ce que j'ai fait

  • Backend Python (FastAPI + instagrapi) pour interagir avec les DMs Instagram
  • Authentification avec sauvegarde de session et gestion des challenges
  • Récupération des conversations, messages texte, images et vocaux
  • Envoi de messages texte et images via l'API non-officielle
  • WebSocket temps réel pour la synchro instantanée montre ↔ backend
  • Frontend Expo (React Native + TypeScript) optimisé écran circulaire
  • Lecture et enregistrement audio pour les messages vocaux (expo-av)
  • Cache local des messages et médias avec expo-file-system

TypeScript React Native Expo Python FastAPI instagrapi WebSocket Wear OS

03 — Épreuve E6

Réalisations professionnelles

Réalisation n°1 · 26/01/2026

Release Notes — Affichage lors des montées de version

Cotranet · Stage 2025–2026 · Seul

Mise en place d'un système qui vérifie, à la connexion, si l'utilisateur a déjà consulté la dernière version de l'application. Si ce n'est pas le cas, une modale affiche les notes de version en Markdown, converties en HTML. La consultation est ensuite enregistrée en base.

Back-end Java : Controller REST, Service avec lecture de fichiers Markdown et fallback linguistique, DTO structuré en Map<String, String>. Front-end Vue.js : service api.ts, typage TypeScript du DTO, modale de rendu Markdown avec sélection automatique de langue.

Réalisation n°2 · 01/11/2025

Documentation API via Swagger (Springdoc OpenAPI)

Cotranet · Stage 2025–2026 · Seul

Intégration de Swagger (Springdoc OpenAPI) dans le back-end Java/Spring Boot pour générer automatiquement une documentation d'API REST interactive, toujours synchronisée avec le code source.

La documentation est exposée via Swagger-UI puis intégrée dans l'application Vue.js dans un composant dédié (iframe), permettant aux développeurs de consulter et tester les endpoints directement depuis l'interface.
